Ты генерируешь синтетический датасет для NER по СТС РФ (свидетельство о регистрации ТС).

Твоя задача в 2 этапа:
ЭТАП 1: сгенерировать реалистичный OCR-подобный текст СТС (как после OCR, с шумом/опечатками).
ЭТАП 2: разметить в этом же образце сущности владельца.

Важно:
- Данные должны соответствовать РФ.
- ФИО в основном российские, но иногда допускай кавказские/тюркские отчества и фамилии (например: оглы, кызы, Асланов, Магомедов, Темирханов и т.п.).
- В тексте должны быть как кириллические ФИО, так и латиница (но только фамилия и имя).
- Номера/серии/даты/адреса/коды генерируй сам, реалистично.

Верни СТРОГО JSON без комментариев, в таком формате:
{
  "samples": [
    {
      "doc_id": "sts_001",
      "variant": "default",
      "text": ".... OCR-like raw text ....",
      "entities": {
        "surname_ru": "",
        "name_ru": "",
        "patronymic_ru": "",
        "surname_en": "",
        "name_en": ""
      }
    }
  ]
}

Правила генерации text:
1) text должен быть похож на OCR-вывод СТС: многострочный, с полями документа.
2) Добавляй типичные блоки: СОБСТВЕННИК(владелец), адрес, особые отметки, дата выдачи, код подразделения.
3) Допускай OCR-шум: пропуски символов, смешение похожих букв (О/0, В/8, И/Й), лишние пробелы, редкие артефакты.
4) В части образцов делай и RU, и EN написание ФИО; в части — только RU; в части — частично.
5) Иногда специально делай сложные случаи: редкие фамилии, двойные фамилии, дефисы.

Правила разметки entities:
1) Размечай ТОЛЬКО то, что реально присутствует в text.
2) Если сущность не найдена или неуверенно читается — пустая строка.
3) RU-поля только кириллица в UPPERCASE.
4) EN-поля только латиница в UPPERCASE.
5) patronymic_ru может быть как классическое (ИЧ/ВНА), так и оглы/кызы.
6) Убирай запятые, точки и лишние символы из entities.

Сгенерируй N samples (N я укажу отдельно). Для каждого doc_id по возможности делай 3 варианта:
- default
- bw
- bw_inv
Смысл вариантов: один и тот же документ, но немного разный OCR-текст/ошибки.

Ниже шаблон структуры OCR-текста СТС (ориентир по реальным полям СТС РФ, особенно оборотная сторона с владельцем):

ШАБЛОН STS_OCR_TEXT (пример каркаса):
[серия/номер СТС]
СОБСТВЕННИК (владелец)
{SURNAME_RU}
{SURNAME_EN}
{NAME_RU}
{NAME_EN}
{PATRONYMIC_RU}
Республика, край, область
{REGION_RU}
Район
{DISTRICT_RU}
Населенный пункт
{CITY_RU}
Улица
{STREET_RU}
Дом {HOUSE} Корпус {BUILDING} Квартира {FLAT}
Особые отметки
{SPECIAL_NOTES}
{DOC_BASIS_LINE_1}
{DOC_BASIS_LINE_2}
Код подразделения {DEPT_CODE}
Дата выдачи {ISSUE_DATE}
Подпись

Доп. поля (можно иногда добавлять в text, как OCR-шумные строки):
- VIN
- Марка, модель ТС
- Год выпуска
- Шасси (рама)
- Кузов (кабина, прицеп)
- Цвет
- Мощность двигателя
- Рабочий объем двигателя
- Разрешенная максимальная масса
- Масса без нагрузки

Ограничения для генерации ФИО:
- {SURNAME_RU}/{NAME_RU}/{PATRONYMIC_RU}: реалистичные для РФ.
- Иногда используй кавказские/тюркские формы отчества: ОГЛЫ, КЫЗЫ.
- EN-поля только как транслитерация фамилии/имени (не отчества).
- В части сэмплов EN-поля могут отсутствовать в тексте.
